<h3>ü´Ç How to collaborate</h3>
<!--add here a step by step on how people can download the repository and make new branches.-->
<h4>1. Fork the repository</h4>
  <ul>
    <li>Navigate to the main repository page on GitHub;</li>
    <li>Click the <strong>Fork</strong> button in the top-right corner;</li>
    <li>This creates a copy of the codebase under your own GitHub account, allowing you to make changes without affecting the original "upstream" project.</li>
  </ul>
<h4>2. Clone and Set Up Locally</h4>
  <ul>
    <li>Open your terminal and run: git clone https://github.com/YOUR-USERNAME/rpg-game-repo.git;</li>
    <li>Move into the project directory: cd rpg-game-repo;</li>
    <li><strong>Pro Tip:</strong> Connect your local copy back to the original source so you can pull updates: git remote add upstream https://github.com/ORIGINAL-OWNER/rpg-game-repo.git</li>
  </ul>
<h4>4. Make changes and commit</h4>
<p>Write your code, add your RPG items, or fix that pesky CSS bug.</p>
  <ul>
    <li>Stage your changes: git add .</li>
    <li>Commit with a clear, descriptive message: git commit -m "Add: Fireball spell logic and D20 roll mechanics"</li>
  </ul>
<h4>Push and Open a Pull Request (PR)</h4>
<p>Once your code is ready, send it back to GitHub:</p>
  <ul>
    <li>Push your branch to your fork: git push origin feature/new-magic-system</li>
    <li>Go to the original repository on GitHub. You will see a banner saying "Compare & pull request."</li>
    <li>Click it, describe your changes in the text box, and submit.</li>
  </ul>
<h4>Code Review and Merge</h4>
  <ul>
  <p>THe project mainteiner will review the code.</p>
      <ul>
        <li><strong>Discussion:</strong> The maintainer might ask for small changes;</li>
        <li><strong>Approval:</strong> Once satisfied, the maintainer clicks MERGE, and the new code becomes part of the main game!</li>
      </ul>
  </ul>
